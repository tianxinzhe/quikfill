# QuickFill 浏览器插件 - 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位
QuickFill 是一款主打 **绝对隐私、纯本地运行、可视化卡片操作** 的 Chrome 侧边栏快捷文本填充工具。

### 1.2 目标用户群体
- **开发测试人员**：高频填写测试数据
- **跨境电商/国内运营客服**：高频回复相同话术
- **日常办公族**：需要快速输入常用文本

### 1.3 核心痛点解决
| 痛点 | 解决方案 |
|------|----------|
| 网页多标签页切换割裂 | Chrome Side Panel 常驻侧边栏 |
| 中文输入法与快捷键冲突 | 可视化卡片点击操作 |
| 现代前端框架（React/Vue）无法直接赋值表单 | 智能注入方案 + 剪贴板兜底 |

---

## 2. 核心用户角色与典型场景

### 2.1 测试工程师 (QA)
- **场景**：造数据或测试表单时，需要反复输入不同的手机号、测试账号和邮箱
- **需求**：一键填满且支持随机变量

### 2.2 电商运营/客服
- **场景**：存有几百条常用话术和商品链接
- **需求**：不需要记忆繁琐的快捷键，看着侧边栏卡片，点一下就能精准回复

### 2.3 重度隐私主义者
- **场景**：工作涉及企业内部敏感系统或报销资料
- **需求**：严禁任何数据上传云端，要求工具 100% 离线安全

## 3. 产品功能清单

### 3.1 核心填充与浏览器交互

**需求描述**：用户在网页中激活任意输入框，点击侧边栏对应的文本卡片，内容必须立刻填入该输入框中。

**验收标准**：
- ✅ 完美兼容基于 React、Vue、Angular 等现代框架构建的网页
- ✅ 填充后不会因双向绑定失效而导致表单提交变空
- ✅ 物理兜底保障：面对极端安全限制的输入框，自动将卡片内容写入系统剪贴板
- ✅ 剪贴板写入后展示视觉提示："已写入剪贴板，请直接 Ctrl+V"

### 3.2 网页划选快速收藏

**需求描述**：用户在浏览任何网页时，遇到高频使用的文本或话术，无需手动复制再打开插件粘贴。

**验收标准**：
- ✅ 用户用鼠标蓝选网页任意文本，点击右键，右键菜单中展示"📥 收藏到 QuickFill"选项
- ✅ 点击后，侧边栏无需显式打开，该文本静默追加保存到本地数据库

### 3.3 离线数据管理

**需求描述**：由于主打无云端服务器、无账号体系的绝对隐私模式，需提供离线备份机制。

**验收标准**：
- ✅ 一键导出：点击后，插件将本地所有文本卡片数据打包为 `.json` 文件并触发浏览器下载
- ✅ 全程不经过任何网络传输
- ✅ 一键导入：用户上传备份的 `.json` 文件后，插件校验文件格式
- ✅ 提供"覆盖导入"或"追加合并"的选项

### 3.4 动态变量占位符

**需求描述**：解决死文本无法应对动态场景的痛点。

**验收标准**：
- ✅ 支持用户在配置卡片文本时，混入特定的语法占位符
- ✅ `{{date}}`：自动转化为当天日期（格式：YYYY-MM-DD）
- ✅ `{{time}}`：自动转化为当前时间（格式：HH:mm:ss）
- ✅ `{{random_phone}}`：自动实时生成符合中国大陆规范的随机 11 位测试手机号

### 3.5 侧边栏视图管理

**需求描述**：当用户存储的卡片变多时，提供高效的过滤、归类和排序手段。

**验收标准**：
- ✅ 轻量实时搜索：侧边栏顶部常驻搜索框，键盘每敲击一个字符，卡片列表实时进行文本关键词过滤
- ✅ 标签分类：允许为每条卡片设置颜色分类标签（如工作、测试、个人）
- ✅ 一键置顶：高频卡片支持点击"图钉"图标，永远固定在侧边栏最上方
- ✅ 手动排序：支持通过拖拽交互和上移/下移按钮自定义卡片顺序

### 3.6 广告变现流

**需求描述**：在不损害用户隐私和核心操作的前提下，利用侧边栏长驻特点进行商业化变现。

**验收标准**：
- ✅ 在侧边栏 UI 的最顶部，留出不影响文本列表操作的固定广告展示区
- ✅ 集成完全适配 Manifest V3 规范的广告组件（如 PlayaYield）
- ✅ 利用海外广告源在后台无感知安全填充

## 4. 非功能性需求与用户体验规范

### 4.1 隐私与安全限制

| 要求 | 说明 |
|------|------|
| 零网络依赖 | 除广告组件外，业务逻辑严禁任何 fetch、XMLHttpRequest 或上传用户数据 |
| 权限最小化 | 仅申请侧边栏、本地存储、当前标签页、右键菜单和剪贴板写入五个核心权限 |

### 4.2 性能与交互体验

| 指标 | 要求 |
|------|------|
| 常驻稳定性 | 采用 Chrome 官方 Side Panel 架构，切换标签页、刷新网页时状态不丢失 |
| 响应速度 | 本地卡片加载、搜索过滤、点击填充反馈时间 ≤ 100ms |

## 5. 技术架构要点

### 5.1 Chrome Extension 核心配置
- **Manifest 版本**：V3（必须）
- **核心权限**：sidePanel、storage、activeTab、contextMenus、clipboardWrite

### 5.2 数据存储方案
- **存储机制**：chrome.storage.local（纯本地，零云端）
- **数据格式**：JSON 结构化存储

### 5.3 关键技术点
1. **表单注入**：针对 React/Vue/Angular 的双向绑定兼容方案
2. **剪贴板操作**：chrome.clipboard API + Fallback 方案
3. **Side Panel 通信**：chrome.runtime 消息机制
